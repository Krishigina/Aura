## 1. Введение

### 1.1 Назначение

Настоящий документ представляет полную спецификацию требований к системе «Aura» — интеллектуальной платформе для персонализированного подбора косметических средств с использованием технологий искусственного интеллекта.

Документ предназначен для:

- Разработчиков системы — в качестве технического руководства
- Тестировщиков — для формирования тест-планов
- Заказчика — для согласования требований и оценки готовности продукта
- Проектных менеджеров — для планирования разработки и ресурсов

### 1.2 Соглашения, принятые в документе

**Терминологические соглашения:**

- «Система», «Платформа», «Aura» — взаимозаменяемые обозначения программного продукта
- «Мобильное приложение» — Android-приложение на Kotlin/Compose
- «Веб-интерфейс» — административная панель на React
- «AI-сервис» — Python-микросервис для ИИ-функций

**Нотационные соглашения:**

- Функциональные требования нумеруются в формате FR-XXX
- Нефункциональные требования нумеруются в формате NFR-XXX
- Приоритеты требований: High (высокий), Medium (средний), Low (низкий)

**Графические соглашения:**

- Диаграммы выполнены в текстовой нотации ASCII
- Модели данных представлены в нотации «сущность — атрибуты»

### 1.3 Границы проекта

**В рамках проекта:**

- Мобильное приложение для платформы Android

**За рамками проекта:**

## 2. Общее описание

### 2.1 Общий взгляд на продукт

**Aura** — это интеллектуальная система подбора косметики, которая помогает пользователям найти оптимальные средства ухода на основе их индивидуальных особенностей кожи, проблем и предпочтений.

**Ценность для пользователя:**

- Персонализированные рекомендации вместо универсальных советов
- ИИ-ассистент для ответов на вопросы об уходе
- Отслеживание использования средств с напоминаниями
- Анализ составов косметики на предмет безопасности

**Бизнес-модель:**

- B2C: Подписка на персонализированные рекомендации
- B2B: API для косметических брендов и ритейлеров

### 2.2 Классы и характеристики пользователей

| Класс пользователя | Описание | Характеристики |
|-------------------|----------|----------------|
| **Конечный пользователь** | Человек, использующий мобильное приложение для подбора косметики | Возраст 18-55 лет, интерес к уходу за кожей, готовность предоставить данные о коже |
| **Администратор** | Сотрудник, управляющий каталогом продуктов и контентом | Доступ к веб-интерфейсу, управление продуктами, просмотр статистики |
| **AI-ассистент** | Интеллектуальный агент в чате | ИИ-модель для ответов на вопросы, генерации рекомендаций |

**Целевые сегменты:**

1. **Молодая аудитория (18-30):** Акцент на профилактике, акне, базовый уход
2. **Зрелая аудитория (30-45):** Акцент на anti-aging, увлажнении, проблемах пигментации
3. **Специализированная аудитория:** Чувствительная кожа, аллергии, специфические проблемы

### 2.3 Операционная среда

**Мобильное приложение:**

- Android 8.0+ (API 26+)
- Kotlin 1.9.x, Compose 1.5.x
- Минимум 3 ГБ ОЗУ, 100 МБ свободного места

**Backend:**

- Python 3.11+
- FastAPI + asyncpg
- PostgreSQL 15+, Redis 7+, Weaviate

**AI-сервис:**

- Python 3.11+
- FastAPI
- LLM (OpenAI API / аналоги)
- Vector DB (Weaviate)

### 2.4 Ограничения дизайна и реализации

**Технические ограничения:**

- Ограниченные ресурсы мобильного устройства
- Зависимость от качества интернет-соединения
- Лимиты AI API (rate limits, стоимость)

**Бизнес-ограничения:**

- Конфиденциальность персональных данных (тип кожи, проблемы)
- Регуляторные требования к косметической информации
- Зависимость от сторонних AI-провайдеров

### 2.5 Предположения и зависимости

**Предположения:**

- Пользователи готовы предоставить данные о своей коже для получения рекомендаций
- Интернет-соединение будет доступно для основных функций
- AI-модель способна предоставлять корректные рекомендации

**Зависимости:**

- OpenAI API (или аналог) для LLM-функций
- Weaviate для векторного поиска
- PostgreSQL для хранения данных
- Внешние источники данных о косметике (парсинг)

## 3. Функции системы

### 3.1 Аутентификация и авторизация (FR-001)

**Описание и приоритет:** High

Система должна обеспечивать безопасную регистрацию и вход пользователей. Реализовано через JWT-токены.

**Функциональные требования:**

- FR-001.1: Регистрация пользователя с именем, email, паролем
- FR-001.2: Вход по email и паролю
- FR-001.3: Автоматическое обновление access-токена
- FR-001.4: Выход из системы с очисткой токенов
- FR-001.5: Хранение токена в безопасном хранилище устройства

### 3.2 Профиль и анкета кожи (FR-002)

**Описание и приоритет:** High

Сбор и хранение информации о типе кожи, проблемах, аллергиях и целях пользователя.

**Функциональные требования:**

- FR-002.1: Анкета с вопросами о типе кожи (сухая, жирная, комбинированная, нормальная)
- FR-002.2: Выбор проблем (акне, пигментация, морщины, чувствительность и др.)
- FR-002.3: Указание аллергенов и противопоказаний
- FR-002.4: Определение целей ухода (увлажнение, anti-aging, осветление и др.)
- FR-002.5: Сохранение анкеты с привязкой к пользователю
- FR-002.6: Возможность редактирования анкеты
- FR-002.7: Пропуск анкеты с ограничением функциональности

### 3.3 Каталог продуктов (FR-003)

**Описание и приоритет:** High

Отображение и фильтрация косметических продуктов из базы данных.

**Функциональные требования:**

- FR-003.1: Получение списка продуктов с пагинацией
- FR-003.2: Поиск продуктов по названию и бренду
- FR-003.3: Фильтрация по типу кожи
- FR-003.4: Фильтрация по назначению (увлажнение, очищение, anti-aging и др.)
- FR-003.5: Просмотр карточки продукта с детальной информацией
- FR-003.6: Отображение процента совпадения с профилем пользователя
- FR-003.7: Плейсхолдер для продуктов без изображения

### 3.4 AI-чат ассистент (FR-004)

**Описание и приоритет:** High

Интерактивный чат с ИИ для консультаций по уходу за кожей.

**Функциональные требования:**

- FR-004.1: Отправка текстовых сообщений ассистенту
- FR-004.2: Получение ответов от AI-модели
- FR-004.3: Быстрые вопросы (шаблоны) для удобства
- FR-004.3: Отображение индикатора «печатает»
- FR-004.4: История сообщений в рамках сессии
- FR-004.5: Отображение времени сообщений
- FR-004.6: Кнопка действия в сообщениях (например, «Открыть камеру»)

### 3.5 Профиль пользователя (FR-005)

**Описание и приоритет:** Medium

Управление личными данными и настройками.

**Функциональные требования:**

- FR-005.1: Отображение имени и email пользователя
- FR-005.2: Просмотр заполненной анкеты
- FR-005.3: Редактирование профиля
- FR-005.4: Переход к заполнению/обновлению анкеты
- FR-005.5: Выход из аккаунта

### 3.6 Уведомления и напоминания (FR-006) [БУДУЩЕЕ]

**Описание и приоритет:** Medium

Система напоминаний об уходе за кожей.

**Функциональные требования:**

- FR-006.1: Напоминания о утреннем/вечернем уходе
- FR-006.2: Напоминания о необходимости пополнения средств
- FR-006.3: Интеграция с системными уведомлениями Android
- FR-006.4: Настройка времени напоминаний

### 3.7 Трекер использования (FR-007) [БУДУЩЕЕ]

**Описание и приоритет:** Medium

Отслеживание использования косметических средств.

**Функциональные требования:**

- FR-007.1: Логирование использования средства
- FR-007.2: Оценка эффективности средства
- FR-007.3: История использования
- FR-007.4: Статистика использования по категориям

### 3.8 Анализ составов (FR-008) [БУДУЩЕЕ]

**Описание и приоритет:** Low

AI-анализ безопасности ингредиентов косметики.

**Функциональные требования:**

- FR-008.1: Разбор состава на ингредиенты
- FR-008.2: Оценка безопасности по базе данных ингредиентов
- FR-008.3: Выявление конфликтующих ингредиентов
- FR-008.4: Рекомендации по совместимости с типом кожи

### 3.9 Административная панель (FR-009) [ЧАСТИЧНО РЕАЛИЗОВАНО]

**Описание и приоритет:** Medium

Веб-интерфейс для управления контентом.

**Функциональные требования:**

- FR-009.1: Управление каталогом продуктов (CRUD)
- FR-009.2: Импорт/экспорт продуктов (CSV)
- FR-009.3: Загрузка изображений продуктов
- FR-009.4: Управление категориями и справочниками
- FR-009.5: Просмотр статистики использования

**Для администраторов:**

- Популярные продукты и категории
- Использование AI-ассистента

### 4.4 Получение, целостность, хранение и утилизация данных

**Получение данных:**

- Пользователи: через UI мобильного приложения
- Продукты: ручной ввод + импорт из CSV + парсинг URL
- Составы: парсинг с косметических сайтов

**Целостность:**

- Валидация на уровне API (Pydantic/FastAPI)
- Ограничения БД (NOT NULL, UNIQUE, FOREIGN KEY)
- Транзакционность для связанных операций

**Хранение:**

- PostgreSQL: основное хранилище
- Redis: сессии, кэш, rate limiting
- Weaviate: векторные представления для RAG и рекомендаций

**Утилизация:**

- Удаление аккаунта по запросу пользователя
- Автоматическая очистка старых сессий
- Ротация логов
